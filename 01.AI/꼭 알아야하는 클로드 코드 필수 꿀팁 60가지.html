  • 헤드리스 모드 실행 (2:17)
    claude -P "프롬프트 내용" 플래그를 사용하여 인터랙티브 모드 없이 클로드 코드를 실행하고 응답을 즉시 받아볼 수 있습니다. 이는 프로그래밍 방식 또는 멀티 에이전트 모드에서 유용합니다.
  • JSON 아웃풋 포맷 (2:43)
    --output-format json 플래그를 추가하여 헤드리스 모드에서 응답을 JSON 형식으로 받아 데이터를 다루기 쉽게 합니다.
  • 로그 디버깅 (Verbose) (3:04)

    자주 사용하는 GitHub 관련, 테스트 관련 등의 명령어를 커스텀 슬래시 커맨드로 미리 만들어 반복적으로 활용할 수 있으며, argument 값을 사용하여 프롬프트를 받을 수도 있습니다.
  • 커스텀 슬래시 커맨드 스코프 조절 (13:31)
    프로젝트 관련 커맨드는 프로젝트 스코프에, 전반적으로 사용하고 싶은 커맨드는 사용자 스코프에 정의하여 중복 정의를 피할 수 있습니다.
  • Explain, Plan, Commit 순서 따르기 (13:50)
    클로드 코드와의 대화 시 충분한 시간을 갖고 프로젝트를 분석(Explain)하고, 작업 계획(Plan)을 세운 후, 작업을 집행하고 커밋(Commit)하는 순서를 따르는 것이 효율적입니다. 문맥이 많고 가이드라인이 명확할수록 더 정확한 작업을 수행할 수 있습니다.
  • Thinking Token 활용 (14:12)
    Think, Think Hard, Think Harder, UltraThink 순서로 더 많은 토큰을 배정하여 클로드 코드가 더 깊이 생각하도록 유도할 수 있지만, 프로세싱 시간이 길어질 수 있음을 유의해야 합니다.
  • TDD(Test-Driven Development) 적극 활용 (14:30)
    클로드 코드에게 예상 Input/Output 값을 기반으로 테스트 코드를 먼저 작성하도록 지시하고, 테스트 실패를 확인한 후 구현을 작성하도록 유도합니다. 테스트를 먼저 커밋하고 테스트를 통과하는 코드를 작성할 때까지 반복하는 것이 좋습니다.

    ※ TDD: Test-Driven Development (테스트 주도 개발)의 약자로, 실제 코드를 작성하기 전에 해당 코드가 잘 작동하는지 검증하는 테스트 코드를 먼저 작성하고, 그 테스트를 통과하는 프로그램을 만드는 개발 방법론.

  • 스크린샷 적극 활용 (14:51)
    Playwright나 Puppeteer 같은 MCP를 사용하여 자동으로 페이지 스크린샷을 찍고 클로드 코드에 주입하여 작업 효율을 높일 수 있습니다.
  • YOLO 모드 (Dangerous Escape Permissions Flag) (15:03)
    Dangerous Escape Permissions Flag를 사용하면 사용자 허락 없이 클로드 코드가 더욱 자유롭게 코딩할 수 있도록 설정할 수 있습니다. 안전을 위해 Docker의 Dev Container 활용을 권장합니다.
  • 클로드 코드를 선생님으로 사용 (15:23)
    새로운 코드베이스를 이해해야 할 때 클로드 코드에게 전반적인 내용부터 세부적인 부분까지 점차 상세하게 질문하여 학습에 활용할 수 있습니다.
  • Prompt는 항상 상세하게 (15:39)
    구체적인 파일명, Edge Case 포함 여부, Mocking 지시 등 명확하고 상세하게 프롬프트를 작성할수록 더 정확한 결과를 얻을 수 있습니다.
  • 이미지 전달 (15:56)

  • 파일 리스토어 (Tab 사용) (16:22)

  • ESC 두려워하지 않기 (16:29)

  • Rollback 명령 (16:45)

  • 체크리스트나 스크래치패드 활용 (17:05)

  • 작업과 검증 분리 (17:21)
    하나의 에이전트로 코딩 작업을 한 후, clear 명령으로 대화 내역을 삭제하거나 다른 서브 에이전트를 생성하여 결과를 검증함으로써 독립된 관점에서 작업을 수행할 수 있습니다.
  • Work Tree 사용 (17:39)
    Work Tree를 사용하면 여러 브랜치 환경을 쉽게 나누고 독립적으로 작업할 수 있습니다. GitHub와 연동하여 여러 작업을 동시에 실행하고 Pull Request로 병합하는 워크플로우를 구축하면 효율적입니다.
