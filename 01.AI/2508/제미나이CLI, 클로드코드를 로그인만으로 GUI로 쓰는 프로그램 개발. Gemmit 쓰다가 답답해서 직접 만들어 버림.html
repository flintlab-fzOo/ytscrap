영상 길이: 00시간10분01초 (10분)

이 영상은 '알렉스 AI'가 기존 Gemini CLI GUI (Gemmit)의 불편함을 해소하기 위해 직접 개발한 '에이전트 허브(Agent Hub)'라는 GUI 애플리케이션에 대해 설명합니다. 에이전트 허브는 다양한 대규모 언어 모델(LLM)들을 API 키 없이 로그인 방식으로 연동하여 사용할 수 있으며, Multi-Agent Collaboration Protocol (MCP) 서버 관리 기능을 통합하여 직관적인 사용자 경험을 제공하는 것을 목표로 합니다. 개발 과정에서 Zen MCP를 활용하여 Claude, GPT, Gemini를 융합한 코딩 작업을 진행했으며, 그 효율성과 뛰어난 결과물에 대해 강조합니다.

내용

[새로운 GUI 개발 동기] 기존 Gemmit의 아쉬움 (00:00:00)

  • Gemmit(Gemini CLI GUI) 사용 후 직관적이지 않다고 느껴 직접 '에이전트 허브' 개발을 시작했습니다.
  • GUI는 직관적이고 편리하며 예뻐야 한다는 생각에 미치지 못하는 기존 GUI에 대한 불만을 언급했습니다.

[에이전트 허브 초기 화면 및 기능 소개] 멀티 LLM 지원 (00:00:20)

  • 초기 화면에는 멀티 LLM (Large Language Model) 지원, Zen MCP 자동화, 로그인 기능이 있습니다.
  • 로그인 후 여러 LLM(Gemini, Claude, GPT)을 선택하여 사용할 수 있는 UI를 제공합니다.
  • API가 아닌 로그인(Auth) 방식으로 LLM을 연결합니다.

[에이전트 허브 UI 구성 및 목표] 직관적인 사용성 강조 (00:01:04)

  • 왼쪽에는 프로젝트 및 대화 히스토리, 중앙에는 대화창, 오른쪽에는 MCP 연결 및 설정 기능이 있습니다.
  • 전체적으로 직관적인 UI를 통해 사용 편의성을 높이는 것을 목표로 합니다.

[Gemini CLI 연동 성공 및 작동 방식] 로그인 기반 연동 (00:01:52)

  • Gemini CLI 로그인 정보를 인식하여 연동되면 파란색으로 표시됩니다.
  • 연결된 Gemini CLI가 질문에 답변하는 모습을 시연했습니다.
  • 초기에는 폰트나 자간 등의 UI 정리가 필요하다고 언급했습니다.
  • Gemini 연동은 매우 잘 되며, API 키 없이 다양한 LLM을 사용할 수 있는 환경을 구축하고 있습니다.
  • 답변에 약간의 시간 지연이 있지만, CLI가 응답하는 형태임을 확인했습니다.

※ CLI: Command Line Interface의 약자로, 사용자가 명령어를 직접 입력하여 컴퓨터를 제어하는 방식의 인터페이스.

※ GUI: Graphical User Interface의 약자로, 그림이나 아이콘을 통해 사용자가 컴퓨터를 직관적으로 조작할 수 있도록 돕는 인터페이스.

※ LLM: Large Language Model의 약자로, 대량의 텍스트 데이터를 학습하여 인간처럼 텍스트를 이해하고 생성하는 인공지능 모델.

※ Auth: Authentication의 줄임말로, 사용자 신원을 확인하고 접근 권한을 부여하는 과정 또는 시스템.

[에이전트 허브의 최종 목표] 통합 관리 시스템 (00:04:51)

  • 왼쪽에는 프로젝트와 히스토리 관리, 가운데는 Gemini, GPT, Claude 엔진을 선택하여 대화.
  • 나아가 이 LLM들이 서로 대화할 수 있는 창구 마련.
  • 오른쪽 MCP 관리 섹션에서는 GitHub에서 MCP 서버를 검색하고, 요약 정보를 확인하여 GUI에서 바로 설치할 수 있도록 세팅할 예정.
  • 3개의 LLM 모델과 수십에서 수백 개의 MCP 서버를 한 창에서 모두 관리하는 것이 최종 목표입니다.

※ MCP: Multi-Agent Collaboration Protocol의 약자로, 여러 에이전트(AI)들이 서로 협력하여 작업을 수행하기 위한 통신 및 협업 규약.

※ GitHub: 소프트웨어 개발 프로젝트를 위한 웹 기반 호스팅 서비스로, 버전 관리, 협업, 코드 공유 등을 지원한다.

[개발 환경 및 Zen MCP 활용] 효율적인 코딩 경험 (00:05:59)

  • 에이전트 허브는 Claude Desktop에 데스크탑 커맨더를 달아 만들었습니다.
  • Zen MCP(Multi-Agent Collaboration Protocol) 서버를 활용하여 Claude, GPT, Gemini를 융합하여 코딩을 진행.
  • Zen MCP는 디버깅, UI/UX 설계, 아키텍처 구성 등 코딩 전반에 걸쳐 뛰어난 성능을 보여줬습니다.
  • 프롬프트 지시를 통해 아키텍처를 짜고, 필요한 기술(Node, React, Electron 등)을 정하며, 역할 분담(GPT, Gemini, Claude)을 효율적으로 수행했습니다.
  • 로그인 시스템은 Firebase 기반으로 구축될 예정입니다.

※ Zen MCP: 영상에서 언급된 Multi-Agent Collaboration Protocol 서버의 한 종류로, 여러 LLM을 융합하여 코딩 및 개발 작업을 효율적으로 수행하는 데 사용됨.

※ Firebase: 구글이 제공하는 모바일 및 웹 애플리케이션 개발 플랫폼으로, 인증, 실시간 데이터베이스, 클라우드 스토리지 등 다양한 백엔드 서비스를 제공한다.

※ React: 사용자 인터페이스를 구축하기 위한 JavaScript 라이브러리로, 단일 페이지 애플리케이션 개발에 널리 사용된다.

※ Electron: 웹 기술(HTML, CSS, JavaScript)을 사용하여 데스크톱 애플리케이션을 만들 수 있게 해주는 오픈 소스 프레임워크.

※ 프론트엔드: 사용자에게 직접 보이는 애플리케이션의 부분으로, UI(사용자 인터페이스)와 UX(사용자 경험)를 담당한다.

[Zen MCP의 활용 권장 및 연동 방법] 개발 효율성 극대화 (00:09:12)

  • 모델링뿐만 아니라 코딩 작업 시에도 Zen MCP를 사용하는 것을 강력히 추천했습니다.
  • Zen MCP 서버는 GitHub에서 검색하여 사용할 수 있으며, Claude 외 다른 LLM을 사용하려면 해당 API를 추가로 연동해야 합니다.

※ API: Application Programming Interface의 약자로, 응용 프로그램에서 사용할 수 있도록 운영 체제나 프로그래밍 언어가 제공하는 기능을 제어할 수 있게 하는 인터페이스.
