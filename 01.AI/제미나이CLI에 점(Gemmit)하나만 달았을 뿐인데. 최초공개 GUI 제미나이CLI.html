제목: 제미나이CLI에 점(Gemmit)하나만 달았을 뿐인데. 최초공개 GUI 제미나이CLI

영상 길이: 00시간15분32초 (15분)

이 영상은 사용자가 AI 간 상호작용(A2A)을 구현하는 과정에서 겪었던 LLM(거대 언어 모델)의 컨텍스트 한계와 CLI(명령줄 인터페이스) 환경의 불편함을 해소하기 위한 탐색 과정을 담고 있습니다. 기존의 클로드 데스크탑, GPT, 젠(Jen) 등의 활용 방안을 비교 분석하고, 최종적으로 GUI(그래픽 사용자 인터페이스) 환경에서 컨텍스트 한계 없이 A2A를 구현할 수 있는 '젠밋(Jenmit)'이라는 솔루션을 소개하며 그 기능과 장점을 시연합니다.

📖 내용

[문제점 인식 및 A2A 시도]

  • 최근 MCP와 젠(Jen)을 통해 클로드, GPT, 제미나이 등 다양한 LLM을 연결하여 A2A(AI to AI) 구현을 시도함 (00:00:00).

    • ※ A2A: AI(인공지능) 모델들이 서로 소통하고 협력하여 작업을 수행하는 방식.

    ※ MCP: Multi-Chat-Platform의 약자로, 여러 인공지능 채팅 모델을 하나의 플랫폼에서 통합하여 사용할 수 있게 해주는 도구.

    ※ LLM: Large Language Model의 약자로, 방대한 텍스트 데이터를 학습하여 사람의 언어를 이해하고 생성하는 능력을 가진 대규모 인공지능 모델.

  • 복잡한 단백질 구조 모델링 작업 시 클로드의 컨텍스트 한계가 크게 느껴짐 (00:00:15).

    • ※ 컨텍스트: LLM에서 대화나 작업의 '맥락' 또는 '기억'을 의미하며, 모델이 현재 처리하는 정보의 범위를 나타냄.

[대안 탐색: CLI 기반 LLM]

  • 클로드만으로는 A2A 구현에 컨텍스트 한계가 있음을 인지하고, 이를 해결하기 위해 커서, 제미나이 CLI, 코덱스 CLI 등을 고려함 (00:00:41).

    • ※ CLI: Command Line Interface의 약자로, 사용자가 명령어를 직접 입력하여 컴퓨터를 제어하는 방식의 인터페이스.

  • 커서는 유료, 안정성, LLM 선택의 제약으로 제외하고, 코덱스 CLI는 명확한 한계가 있었음 (00:01:01).
  • 결국 대화형 코드 및 데스크탑 제어를 위해 제미나이 CLI가 가장 적합한 대안이라고 판단함. 제미나이 CLI는 컨텍스트 한계가 없음 (00:01:18).

📊 각 LLM 및 구성 방식 비교

[Claude Desktop + MCP] (00:01:50)

  • 장점: GUI(그래픽 사용자 인터페이스) 사용 가능.

    • ※ GUI: Graphical User Interface의 약자로, 아이콘이나 메뉴 등 시각적 요소를 통해 사용자가 컴퓨터와 상호작용하는 방식의 인터페이스.

  • 단점: 컨텍스트가 짧아 복잡한 작업이 어려움. 대화가 길어지면 새 대화창을 열어야 함.
  • 구동료: 프로 플랜 기준 20달러.

[Claude Desktop + GPT + MCP] (00:03:03)

  • 장점: 클로드와 GPT 두 LLM 연결로 복잡한 작업 및 성능 향상 가능.
  • 단점: 여전히 짧은 컨텍스트 한계 존재. LLM 간 통신 오류 빈발 (특히 윈도우 환경).
  • 구동료: 클로드 프로 플랜 + GPT 플러스 플랜 기준 40달러 (구독료 더블).

[젠(Jen) + API] (00:03:34)

  • 장점: 복잡한 작업 가능, A2A 안정적 처리.
  • 단점: 긴 컨텍스트 소모, LLM 간 통신 시간 소요, 컨텍스트 유지 어려움.
  • 구동료: 클로드 구독료 20달러 + API 비용(예: GPT API) 추가 발생.

    • ※ API: Application Programming Interface의 약자로, 운영체제나 라이브러리 등이 제공하는 기능을 응용 프로그램에서 사용할 수 있도록 만든 인터페이스.

[Gemini CLI + MCP] (00:04:32)

  • 장점: 긴 컨텍스트 수용.
  • 단점: 하나의 LLM만 연결되어 복잡한 작업은 어려움 (A2A 방식이 아님). CLI 환경이라 답답하고 사용에 불편함.
  • 구동료: 하루 1,000 리퀘스트까지 거의 무료.

[Gemini CLI + Jen + MCP] (00:06:17)

  • 장점: 복잡한 A2A 기능 및 성능 향상.
  • 단점: 여전히 CLI 환경이라 사용이 불편함. API 비용 추가 발생.

🖥️ 사용자 경험: GUI 선호

  • CLI 환경(클로드 코드, 제미나이 CLI 등)이 비개발자에게는 매우 답답하게 느껴짐 (00:04:56).
  • GUI 환경에서 코딩 가능한 클로드 데스크탑의 '데스크탑 커맨더' 방식을 선호함 (00:05:37).
    • 토큰 소모는 많지만 간편하게 사용할 수 있다는 장점 때문.

✨ 최적의 대안: Jenmit 발견

  • 제미나이 CLI를 GUI 환경에서 사용하기 위한 방법을 탐색하던 중 GitHub에서 '젠밋(Jenmit)'을 발견함 (00:06:50).
  • 젠밋은 API 키 없이 제미나이 CLI 로그인 정보만으로 연동이 가능하여 GUI 환경에서 제미나이 CLI 사용 가능 (00:07:08).
  • 젠밋에 젠(Jen)이나 다른 MCP를 붙이면 컨텍스트 한계 없이 GUI 환경에서 A2A 기능을 활용 가능하며, API 비용만 지불하면 됨 (00:07:17).
  • 젠밋에 MCP만 연결할 경우, 클로드 데스크탑의 단점(짧은 컨텍스트, 구동료)을 해소하면서 무료로 긴 컨텍스트 환경을 활용 가능 (00:07:39).

🚀 Jenmit (잼밋) 상세 분석

  • 어플리케이션 특징: 데스크탑에 설치하여 사용하는 어플리케이션으로, Google Gemini CLI 기반으로 전적으로 실행되며 API 키가 100% 로컬 환경에서 제공됨 (00:08:11).
  • 모바일 연동: '젠밋 앱(Jenmit App)'을 통해 X4 환경에서 모바일로도 제미나이 CLI를 가동시켜 AI 편집 및 프로젝트 관리가 가능함 (00:08:45). MCP를 젠밋에 연결하면 휴대폰으로도 MCP 가동 가능.
  • 주요 기능:
    • AI 기반 채팅 인터페이스 (00:09:24).
    • 실시간 프로젝트 스카폴딩(Scaffolding) 시각화.

      • ※ 스카폴딩: 소프트웨어 개발에서 프로젝트의 기본적인 구조나 뼈대를 자동으로 생성해 주는 기능.

    • 앱 내 파일 탐색, 업데이트, 삭제 등 CLI의 기본 기능 지원 (00:09:32).
    • 클로드 데스크탑의 데스크탑 커맨더처럼 컴퓨터 제어 가능 (00:09:40).
    • MCP(Multi-Chat-Platform) 지원 (00:09:49).
  • 장점: 클로드 데스크탑의 모든 기능에 MCP 연동까지 가능하면서도, 구독료가 없고 컨텍스트 한계가 없음 (00:09:50).
  • 호환성: 맥OS, 윈도우, 리눅스 등 모든 플랫폼에서 작동하는 크로스 플랫폼 지원 (00:10:06).

⚙️ 설치 및 구동 환경

  • 아키텍처: 데스크탑, 서버, 앱, 스크립트 가이드라인으로 구성 (00:10:20).
  • 필요 요소: Node.js, Python, 제미나이 CLI가 설치되어 있어야 함 (00:10:34).
  • 자동 연동: 제미나이 CLI를 한 번 설치하고 로그인해 놓으면, 젠밋이 자동으로 이를 인식하여 연동됨 (00:10:42).
  • 젠밋을 가동할 때 별도로 제미나이 CLI를 켜놓을 필요는 없음 (00:12:23).

🔧 MCP 연동 방법

  • 젠밋 프로젝트 내 `jemmini` 폴더 안에 `setting.json` 파일을 생성하여 MCP 설정 가능 (00:11:04).
  • 기존 클로드 데스크탑의 `settings.json`과 거의 동일한 문법을 사용하지만, 일부 경로 설정 (`tem github` 등)에서 차이가 있음 (00:11:45).

    • ※ JSON: JavaScript Object Notation의 약자로, 데이터를 저장하고 교환하기 위한 경량 데이터 교환 형식.

📺 Jenmit 데모 및 사용자 인터페이스

  • 젠밋을 가동하면 초기 화면이 매우 예쁘게 나타남 (00:12:57).
  • 시작(Start) 버튼을 누르면 이전에 제미나이 CLI와 대화했던 내역을 볼 수 있고, 오른쪽에 디렉토리들을 스카폴딩하여 보여줌 (00:13:00).
  • 좌측은 대화창, 우측은 코딩 프로젝트를 스카폴딩하여 보여주는 영역으로 분리됨 (00:13:16).
  • 프론트엔드 서버를 돌려 테스트 및 디버그 가능 (00:13:41).
  • 주로 바이브 코딩 툴(Vibe Coding Tool) 형태로 활용되며, GUI 환경으로 간주될 수 있어 편리함 (00:13:52).

💬 Jenmit과 대화 예시

  • "안녕"이라고 질문 시, 제미나이 CLI와 연결되어 응답함 (00:13:58).
  • 젠밋은 AI 기반 소프트웨어 개발 프로젝트 관리 도구로, 사용자 목표 명확화, 요구사항 탐색, 설계, 세부 사고 분할 등 프로젝트 초기부터 반복까지 전 과정을 지원한다고 스스로 소개함 (00:15:08).
