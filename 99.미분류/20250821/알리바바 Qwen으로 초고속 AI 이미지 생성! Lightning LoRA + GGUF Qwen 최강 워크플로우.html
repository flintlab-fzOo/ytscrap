개요

이 영상은 알리바바의 AI 이미지 생성 모델인 Qwen을 활용하여 초고속으로 이미지를 생성하는 방법을 설명합니다. ComfyUI 최신 버전 업데이트 후 발생할 수 있는 오류 해결 방법부터 Qwen 모델과 Lightning LoRA, GGUF 모델을 사용하여 텍스트-투-이미지(Text-to-Image)를 생성하고, 생성된 이미지의 디테일을 향상시키는 다양한 후처리 워크플로우(Qwen, Flux Crea, Wan 2.2)를 비교 분석합니다. 특히 Qwen이 영어 및 중국어 프롬프트 처리에 뛰어난 장점을 가지고 있음을 강조하며, 빠르고 고품질의 이미지 생성 전략을 제시합니다.

내용

[서론] Qwen 모델 소개 (00:00:00,640)

  • Qwen은 알리바바의 다른 AI 팀에서 개발한 이미지 생성 모델로, Wan(영상 생성 모델)과 유사하게 Flux 계열의 모델들을 개발하고 있음. (00:00:03,580)
  • 향후 이미지 편집 기능(Flux Context와 유사)이 출시될 예정이며, 본 영상에서는 Text-to-Image 생성 및 자연스러운 이미지 후처리 방법을 다룸. (00:00:33,560)

[오류 해결] ComfyUI 업데이트 후 발생하는 에러 해결 방법 (00:00:52,420)

  • ComfyUI 업데이트 시 PyTorch 버전이 2.8로 변경되면서 Triton, Sage Attention 등에서 에러가 발생할 수 있음. (00:01:11,080)
  • 해결책:
    • CMD 실행 경로: ComfyUI 매트릭스 사용자는 packages/ComfyUI/VEMV/Scripts 폴더에서, 포터블 사용자는 python_embedded 폴더에서 CMD 창을 열어야 함. (00:02:00,600)
    • Sage Attention 재설치: 기존 Sage Attention을 삭제(pip uninstall sage-attention) 후, PyTorch 버전(2.8) 및 CUDA 버전(12.8), Python 버전(3.10)에 맞는 최신 버전을 수동으로 설치. (00:02:53,380)
    • Triton 재설치: 기존 Triton을 삭제(pip uninstall triton-windows) 후, PyTorch 버전(2.8)에 맞는 최신 Triton 윈도우즈 버전을 수동으로 설치. (00:03:19,860)
    • Nunchaku 업데이트: ComfyUI/custom_nodes 폴더에서 기존 Nunchaku 폴더를 삭제하고, GitHub에서 최신 버전으로 git clone하여 재설치 후, install_web.json 파일 실행을 통해 허깅 페이스 버전 0.3.2를 설치. (00:08:51,740)
    • 기타 패키지 버전: Xformers 0.0.32 post2, Diffusers 0.4.0, Transformers 4.5.2 최신 버전 사용 권장. (00:11:00,420)
    • ComfyUI 실행 시 로그 확인: 설치된 Python, VRAM, PyTorch, CUDA, Xformers, 그래픽카드 정보 등을 로그에서 확인하여 설치 여부 검증. (00:08:05,380)
*참고*:
  • PyTorch: 페이스북이 개발한 오픈소스 머신러닝 라이브러리. 딥러닝 모델 개발에 널리 사용됩니다.
  • CUDA: NVIDIA가 개발한 병렬 컴퓨팅 플랫폼. GPU를 사용하여 복잡한 계산을 빠르게 처리할 수 있게 합니다.
  • Triton: 오픈AI가 개발한 고성능 머신러닝 컴파일러. GPU에서 딥러닝 모델의 연산을 효율적으로 실행하도록 돕습니다.
  • Sage Attention: 특정 어텐션 메커니즘을 지칭하는 것으로 보이나, 일반적인 용어는 아님. 영상 내에서 ComfyUI 노드 이름으로 언급됨.
  • Nunchaku: ComfyUI의 커스텀 노드 중 하나로, 이미지 생성 워크플로우에 다양한 기능을 추가합니다.
  • PIP: Python Package Installer의 약자로, 파이썬 패키지를 설치하고 관리하는 도구입니다.

[모델 및 LoRA 다운로드/설치] (00:12:30,100)

  • Qwen 이미지 모델: Qwen 이미지 Distil GGUF 모델(Qwen-Image-Distill-Q4KS)을 다운로드. VRAM이 충분하면 Q8, 부족하면 Q4 버전을 사용. (00:15:05,200)
    • 설치 경로: ComfyUI Models 폴더 내 GGUF 또는 DiffusionModels 폴더에 Queen이라는 서브 폴더를 만들어 저장. (00:15:34,420)

    • 설치 경로: ComfyUI Models 폴더 내 LoRas 폴더에 Queen 서브 폴더를 만들어 저장. (00:14:39,760)
  • 텍스트 인코더 및 VAE: Qwen용 텍스트 인코더(7B FP8 Scaled)와 VAE(Queen Image VAE)를 다운로드하여 각각 Models/text_encoder 및 Models/VAE 폴더에 저장. (00:13:00,160)
  • ComfyUI 모델 목록 새로고침: 모델을 추가한 후 키보드 R 키를 눌러 ComfyUI 모델 목록을 업데이트해야 함. (00:16:05,820)
*참고*:
  • LoRA (Low-Rank Adaptation): 대규모 인공지능 모델의 성능 저하 없이 효율적으로 미세 조정할 수 있는 기술입니다. 적은 양의 데이터를 사용해 모델을 특정 작업에 맞게 최적화할 수 있습니다.
  • GGUF: LLM (Large Language Model) 파일을 위한 새로운 형식으로, 다양한 하드웨어에서 모델을 더 쉽게 실행할 수 있도록 최적화되어 있습니다.
  • VAE (Variational AutoEncoder): 잠재 공간에서 이미지를 인코딩하고 디코딩하는 데 사용되는 신경망. 생성된 이미지의 디테일과 선명도를 향상시킵니다.
  • Text Encoder: 텍스트 프롬프트를 AI 모델이 이해할 수 있는 숫자의 벡터(임베딩)로 변환하는 부분입니다.
  • VRAM (Video Random Access Memory): 그래픽 카드에 탑재된 메모리로, AI 모델을 실행하고 이미지를 생성하는 데 필수적입니다. 높을수록 더 크고 복잡한 작업을 처리할 수 있습니다.

[Qwen 워크플로우 구성 및 이미지 생성] (00:16:24,620)

  • 기본 워크플로우: Browse Templates > Image > SWEN Image Text2Image를 기반으로 사용. (00:16:24,620)
  • 노드 연결: Empty Latent Image, Anything Everywhere (텍스트 프롬프트 관리), UNEN Loader GGF (Qwen 모델 로드), FLIP Text Encoder, Model Sampling Aura Flow (Shift 값 3.10), Lora Loader (Lightning LoRA 4-step V1.0), VAE Decode 등을 연결. (00:16:29,520)
  • 프롬프트 처리: Google 자동 번역 노드를 사용하여 한글 프롬프트를 영어로 자동 번역하여 사용. (00:18:31,500)
  • K-Sampler 설정:
    • CFG (Classifier Free Guidance): 1 (Lightning 버전에 최적화)
    • B Noise (Beta Noise): 1
    • Steps: 4 Step (초고속 생성)
    • Sampler: Euler (기본 스케줄러)
    • D Noise (Denoising Strength): 0.3 (후처리 시 사용)
  • VRAM 관리: Clear VRAM, Clear Cache 노드를 사용하여 이미지 생성 시마다 메모리 초기화. (00:19:28,840)
  • 4 Step 이미지 생성 결과: 약 35초 소요. 이미지가 빠르지만 약간의 격자무늬(grid pattern)가 보일 수 있음. (00:20:06,820)
  • 6 Step 이미지 생성 결과: 약 51초 소요. 격자무늬가 덜하지만 완전히 사라지지는 않음. (00:21:08,400)
*참고*:
  • CFG (Classifier Free Guidance): 프롬프트가 이미지 생성에 미치는 영향의 강도를 조절하는 값입니다. 값이 높을수록 프롬프트를 엄격하게 따르지만, 너무 높으면 이미지가 부자연스러워질 수 있습니다.
  • Sampler (샘플러): 노이즈에서 이미지를 점진적으로 생성하는 방식을 정의하는 알고리즘입니다. Euler, DPM++ SDE, DDIM 등 다양한 종류가 있습니다.
  • Scheduler (스케줄러): 샘플러의 진행 방식을 조절하여 이미지 생성 과정에 영향을 미칩니다.
  • Latent Image: 고차원의 픽셀 이미지 데이터를 압축하여 저차원 공간에 표현한 것입니다. 이미지 생성 과정에서 모델은 이 잠재 공간에서 작업합니다.
  • DNoise (Denoising Strength): 이미지를 얼마나 많이 변화시킬지 결정하는 값입니다. 값이 높을수록 원본 이미지에서 크게 변형될 수 있습니다.

[생성된 이미지 디테일 향상 (후처리)] (00:21:49,120)

  • 세 가지 후처리 방식 비교:
    1. Qwen으로 재처리: 생성된 이미지를 Qwen 모델로 D Noise 0.3, 4 Step으로 다시 생성. 약 35초 소요. 격자무늬가 사라지고 선명해짐. (00:23:53,940)
    2. Flux Crea로 재처리 (Nunchaku 사용): Flux Crea 모델을 사용하여 D Noise 0.3, 8 Step으로 재처리. 약 15초 소요. 격자무늬가 완전히 사라지고 피부 표현 등 실사 같은 느낌이 강화됨. (00:24:36,400)
      • Cache/Hold 노드: Nunchaku의 Cache/Hold 노드(블루베어 개발)를 0.1~0.5 범위로 설정하여 스텝을 건너뛰어 속도 향상 가능. (00:25:23,760)
    3. Wan 2.2로 재처리 (Lightning 버전, NAG 사용): Wan 2.2 Lightning 버전을 사용하여 D Noise 0.3, 4 Step으로 재처리. 약 33초 소요. 이미지 선명도 및 디테일이 향상되지만 피부 표현은 Flux Crea가 더 자연스러움. NAG(Negative Attention Guidance) 노드를 활용하여 강력한 네거티브 프롬프트 적용 가능. (00:26:17,120)
*참고*:
  • NAG (Negative Attention Guidance): 부정적인 프롬프트(예: "bad anatomy")에 모델이 더 강력하게 반응하도록 하여 원치 않는 요소를 효과적으로 제거하는 기법입니다.
  • IP Adapter (Image Prompt Adapter): 이미지 프롬프트를 사용하여 모델의 생성 방향을 제어하는 어댑터입니다.

[Qwen의 프롬프트 처리 능력 테스트] (00:27:41,880)

  • 언어 지원: Qwen은 영어와 중국어 프롬프트 처리에 뛰어나며, 글자 생성 능력이 매우 우수함. 한글은 아직 정확하게 지원하지 않음. (00:27:49,920)
  • 글자 생성 테스트: "Confui Sanai Labs" 한글 프롬프트 입력 시, 구글 번역을 통해 영어로 변환되어 대형 전광판에 정확하게 표시됨. (00:30:42,400)
  • 복합 프롬프트 테스트: 12개의 동물 봉제인형을 특정 순서와 배열(4개씩 3줄)로 생성하는 복합 프롬프트 입력 시, Qwen이 순서와 위치, 수량까지 정확하게 따르는 것을 확인. (00:28:45,120)
  • 복잡한 풍경 및 글자 테스트: "한국 얼짱 얼굴", "긴 생머리", "사람 많은 거리", "한국 길거리 음식", "귀여운 동물/캣몬스터 인형", "포켓몬 간판", "분식 간판" 등 복합적인 프롬프트 입력 시, 세부적인 요소들을 잘 반영하여 이미지 생성. 간판의 한글(분식)은 영어(SNACK)로 변환되어 출력됨. (00:32:06,460)

[결론 및 향후 계획] (00:33:59,080)

  • 추천 워크플로우: Qwen으로 4 Step 고속 이미지 생성 → Flux Crea로 디테일 작업(15초 소요)을 통해 빠르고 고품질의 이미지 획득. (00:27:29,380)
  • Qwen Image-to-Image: 현재 Qwen 모델은 Text-to-Image 기반이며, Image-to-Image에 최적화된 Qwen 모델(Flux Context와 유사한 편집 기능)이 출시될 예정이므로 그때 다시 자세히 소개할 예정. (00:34:09,480)
  • 워크플로우 공유: 본 영상의 워크플로우는 멤버십 구글 드라이브 폴더에 업로드 예정. (00:35:14,780)
